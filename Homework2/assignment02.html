Part 1: Linear Regression¶

Linear regression fits an affine model $\mathbb{R}^d \rightarrow \mathbb{R}$ of the form

$$ \hat{y} = b + \mathbf{w}^T \mathbf{x} $$

to a set of $n$ pairs

$$ T = \{(\mathbf{x}_1, y_1), \ldots (\mathbf{x}_n, y_n)\} $$

where $\mathbf{x}_i\in\mathbb{R}^d$ and $y_i\in\mathbb{R}$.

More specifically, it finds a real number $\hat{b}$ and a real vector $\hat{\mathbf{w}}\in\mathbb{R}^d$ that minimizes the average over all pairs in $T$ of some measure of discrepancy between the values $\hat{y}_i = \hat{b} + \hat{\mathbf{w}}^T \mathbf{x}_i$ predicted by the model and the actual values $y_i$:

$$ \hat{b}, \hat{\mathbf{w}} = \arg\min_{b, \mathbf{w}} \frac{1}{n} \sum_{i=1}^n \rho(\phi_i) \;\;\;\mbox{where}\;\;\; \phi_i = b + \mathbf{w}^T \mathbf{x}_i - y_i\;. $$

The quantity $\phi_i$ is called the $i$-th residual and $\rho(\phi_i)$ is the $i$-th loss. Please pay attention to this distinction.

The average discrepancy is a function of both the parameters $b, \mathbf{w}$ of the model and the data points in $T$. To make this dependency explicit we first bundle the model parameters into a single vector $\mathbf{z} = (b, w_1, \ldots, w_d)^T$ and then call the average discrepancy defined above the empirical risk $L_T$:

$$ L_T(\mathbf{z}) = \frac{1}{n} \sum_{i=1}^n \rho(\phi_i(\mathbf{z})) \;. $$

With this notation, we can rewrite the linear regression problem more simply as follows:

$$ \hat{\mathbf{z}} = \arg\min_{\mathbf{z}} L_T(\mathbf{z}) \;. $$

You have studied in previous courses how to solve this minimization problem when

$$ \rho(\phi) = \phi^2 \;. $$

In this case, the regression problem is called Ordinary Least Squares (OLS). The empirical risk is a quadratic function of $\mathbf{z}$ in this case, and the OLS problem can be solved by finding the minimum-norm solution to the linear system

$$ X\mathbf{z} = \mathbf{y} \;. $$

Here, $X$ is an $n$ by $d+1$ matrix and $\mathbf{y}$ is an $n$-dimensional vector:

$$ X = \left[\begin{array}{cc} 1 & \mathbf{x}_1^T \\ \vdots & \vdots \\ 1 & \mathbf{x}_n^T \end{array}\right] \;\;\;\mbox{and}\;\;\; \mathbf{y} = \left[\begin{array}{c} y_1 \\ \vdots \\ y_n \end{array}\right] \;. $$

Problem 1.2 (Exam Style Except for the Code)¶

For later reference, we rewrite the quadratic risk $$ L_T(\mathbf{z}) = \frac{1}{n} \sum_{i=1}^n \rho(\phi_i(\mathbf{z})) $$ as follows: $$ L_T(\mathbf{z}) = \text{mean}(\rho(\boldsymbol{\varphi}_T(\mathbf{z}))) $$ where $\boldsymbol{\varphi}_T(\mathbf{z})$ is a vector containing the $n$ residuals $ \phi_i(\mathbf{z}) \;. $

Applying the function $\rho$ to a vector means to apply it to each of the vector's entries, and therefore the quantity $\rho(\boldsymbol{\varphi}_T(\mathbf{z}))$ is itself a vector with $n$ entries.

In this way, the gradient $\nabla L_T(\mathbf{z})$ of $L_T(\mathbf{z})$ can be written in two steps:

First compute the Jacobian of $\boldsymbol{\varphi}_T(\mathbf{z})$ with respect to $\mathbf{z}$, defined as the following $n\times (d+1)$ matrix: $$ J_T(\mathbf{z}) = \left[\begin{array}{cc} \frac{\partial \phi_1}{\partial z_0} & \ldots & \frac{\partial \phi_1}{\partial z_d} \\ \vdots & & \vdots \\ \frac{\partial \phi_n}{\partial z_0} & \ldots & \frac{\partial \phi_n}{\partial z_d} \end{array}\right] $$ where $\boldsymbol{\varphi}_T(\mathbf{z}) = (\phi_1,\ldots, \phi_n)^T$.

Then, the gradient $\nabla L_T(\mathbf{z})$ of $L_T(\mathbf{z})$ can be easily computed from $\boldsymbol{\varphi}_T(\mathbf{z})$ and $J_T(\mathbf{z})$.

Part 2: Robust Hyperplane Fitting¶

Let us now generalize and modify the simple scenario examined in Part 1 in several directions.

A More Symmetric Hyperplane Equation¶

In Part 1, a hyperplane in $\mathbb{R}^{d+1}$ is the graph of an affine function. This is often awkward, because it requires singling out a special dimension (the $y$ dimension, or the ordinate of the graph), and measure the distance of the data points to the hyperplane in that dimension.

A simple situation in which this causes difficulties is when $d = 1$ and the best fitting line is vertical (that is, parallel to the $y$ axis). In that case, the best solution cannot be expressed in the form $y = b + wx$. It is then better to express the line in the form

$$ \mathbf{n}^T\mathbf{x} = c $$

where $\mathbf{x}$ is in $\mathbb{R}^2$ and contains both abscissa and ordinate. The vector $\mathbf{n}\in\mathbb{R}^2$ has unit norm and is orthogonal to the line, and $|c|$ is the Euclidean distance of the line from the origin of the plane.

For instance, a line with equation

$$ y = \frac{3}{4} x + 1 $$

can be rewritten as follows, after rearranging terms:

$$ -\frac{3}{5} x + \frac{4}{5} y = \frac{4}{5} $$

that is,

$$ \mathbf{n}^T\mathbf{x} = c $$

where

$$ \mathbf{x} = (x, y)^T \;\;\;\mbox{and}\;\;\; \mathbf{n} = \frac{1}{5} (-3, 4)^T \;\;\;\mbox{and}\;\;\; c = \frac{4}{5} \;. $$

Note that $\|\mathbf{n}\| = 1$.

With this representation there is no privileged coordinate (there are two $x$-s rather than an $x$ and a $y$), and all lines on the plane can be represented in this form, including vertical ones.

We redefine $d$ so that $d = 2$ in this case.

In more dimensions, the same equation holds, except that now $\mathbf{n}$ and $\mathbf{x}$ are in $\mathbb{R}^d$ and instead of lines we have hyperplanes.

The "training set" is now just a set of points in $\mathbb{R}^d$:

$$ T = \{\mathbf{x}_1, \ldots, \mathbf{x}_n\} \;. $$

The parameter vector is $\mathbf{z}^T = (c, \mathbf{n})$, and the optimization problem is a constrained one

$$ \hat{\mathbf{z}} = \arg\min_{\mathbf{z} = (c, \mathbf{n})\ :\ \|\mathbf{n}\| = 1} L_T(\mathbf{z}) \;. $$

Residuals¶

The residuals are now

$$ \phi_i = \mathbf{n}^T\mathbf{x} - c \;. $$

Algebraically, these residuals are still the discrepancies between left-hand and right-hand sides of the line equation.

However, with the new formulation these residuals have a different geometrical interpretation. Specifically, $|\phi_i|$ is the Euclidean distance of point $\mathbf{x}_i$ from the line. Let us call these residuals the orthogonal distances of the points from the line.

Robust Loss¶

If the set $T$ contains outliers, that is, points that are not close to any hyperplane that is close to many of the other points, then any one outlier can skew the OLS solution by an arbitrary amount by moving that point far enough from all the others. In the theory of robust estimation, one says that the breakpoint of OLS is one.

To make the problem less sensitive to outliers, one can replace the square (that is, $\rho(\phi) = \phi^2$) by a robust loss function. A simple example is Tukey's loss function

$$ \rho(\phi) = \left\{\begin{array}{ll} \frac{\tau^2}{6} \left[1 - \left(1 - \frac{\phi^2}{\tau^2} \right)^3 \right] & \mbox{if $|\phi| \leq \tau$} \\ \frac{\tau^2}{6} & \mbox{otherwise.} \end{array}\right. $$

def tukey(u, tau):
    inside = np.abs(u) <= tau
    tau2 = tau ** 2
    tau4 = tau2 ** 2
    u2 = u ** 2
    u4 = u2 ** 2
    u6 = u4 * u2
    out_value = tau ** 2 / 6
    in_values = u2 / 2 - u4 / 2 / tau2 + u6 / 6 / tau4
    rho = out_value * ~inside + in_values * inside
    return rho

Lack of Convexity¶

With the new formulation and the more complex data distribution, the risk is no longer necessarily a convex function of the hyperplane parameters in $\mathbf{z}$, and may have local minima, maxima, and saddle points.

Nonconvexity derives from two sources:

  • Tukey's loss function is nonconvex.
  • Data points concentrated along two lines yield at least two local minima, one per line.

It is instructive for you to browse the code. In particular, note the unusual import for numpy:

The module autograd.numpy is a reimplementation of most of numpy to include automatic differentiation. This means that if you implement your functions with this module rather than the standard numpy, you can then let the module itself define new functions that compute the gradients of your functions. For instance, you will later provide a function that computes the Tukey losses for a set of points. This function is bound to the parameter losses of the function run_sgd in sgd.py. If you look at the definition of run_sgd, you will see that this function defines a function risk that computes the mean of your losses, and then you will see the call

risk_gradient = gradient(risk)

This is all you need to do to make a brand new function risk_gradient that, when called, computes the gradient of risk. Rather easy, no?

All of the main packages for deep learning use automatic differentiation. While this slows down the code considerably, because many things need to happen under the hood for automatic differentiation to work its magic, package users do not need to compute derivatives by hand, thereby removing one of the main sources of bugs in deep learning implementations.

Constrained Optimization¶

The constraint $\|\mathbf{n}\| = 1$ in the minimization problem defined above is handled in a straightforward manner in the function step defined in sgd.py. Specifically, a regular step is first taken along the negative gradient of the risk computed on a mini-batch:

$$ \mathbf{z}_{k+1} =\mathbf{z}_k - \alpha \nabla L_B(\mathbf{z}_k) $$

where $B$ is the minibatch (a subset of $T$).

The resulting vector $\mathbf{z}_{k+1} = (c_{k+1}, \mathbf{n}_{k+1})$ is then divided by the Euclidean norm of $\mathbf{n}_{k+1}$. This division has the effect that $\mathbf{z}_{k+1}$ now satisfies the constraint. This technique is called the projected gradient technique. Please read the code of step in sgd.py.

The Decorator¶

The decorator class CallCounter has been imported for you earlier from sgd.py. You do not need to understand Python decorators in detail to use this feature. Suffice it to say that the decoration will keep track of the cumulative number of points in $\mathbb{R}^d$ on which the function tukey_losses is called as SGD is run. This count is an estimate of the computational cost of SGD.

Your Task¶

Show your code for tukey_losses (remember to include the decorator). Then pick a batch size and a learning rate and run run_sgd as follows:

batch_size, learning_rate = ...
run_sgd(tukey_losses, points, batch_size, learning_rate)

The points array was downloaded for you earlier and the function run_sgd was imported earlier as well (in the paragraph on Stochastic Gradient Descent and Automatic Differentiation).

If you look at the definition of run_sgd in sgd.py, you will see that it has a named parameter budget set to int(3e6) (three million) by default. With this value, SGD will terminate if tukey_losses is called on more than that many points overall (see the discussion of the decorator CallCounter above for this count.) So you can consider this value as the total computational budget allowed for this computation.

You will get full credit if you find a batch size and a learning rate for which SGD converges to either true line in points within the given budget. Convergence is achieved when the Euclidean distance between the true value $\mathbf{z}^\ast$ and the value $\hat{\mathbf{z}}$ found by SGD is less than 0.1 . Parameter values for the true lines were given earlier, and SGD is supposed to converge to either of them.

The function run_sgd gives the following diagnostic information:

  • It print out the batch size, learning rate, and total computational expense.
  • It plots a fraction of all the data points and the best-fit line. The title of that plot also gives the line parameters and the error. This is the error that has to go below 0.1 .
