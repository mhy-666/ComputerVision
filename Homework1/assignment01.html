Matrices in LaTeX¶

One way to render

$$ X = \left[\begin{array}{rrr} 1 & -2 & 3 \\ -4 & 5 & -6 \end{array}\right] $$

in math mode (that is, between single or double dollar signs), is to write

X = \left[\begin{array}{rrr}
    1 & -2 & 3 \\
    -4 & 5 & -6
\end{array}\right]

When matrices are row vectors, you can choose this same format, or just use brackets (or parentheses) and commas. That is, all of the following formats are acceptable:

$$ Z = \left[\begin{array}{rrr} 1 & -2 & 3 \end{array}\right] \;\;\;,\;\;\; Z = [1, -2, 3] \;\;\;,\;\;\; Z = (1, -2, 3) $$

Part 2: Differentiation Basics¶

Consider the following toy image:

$$ I = \left[\begin{array}{ccccc} 0 & 24 & 16 & 40 \\ 24 & 32 & 40 & 48 \\ 48 & 32 & 16 & 0 \\ 16 & 16 & 16 & 16 \end{array}\right] \;. $$

Coordinate Notation¶

To avoid confusion with coordinates we index pixels in this image as $I(r, c)$, where $r$ is the row index and $c$ is the column index, consistently with standard matrix indexing conventions. In this part it is not relevant whether $r$ and $c$ start at 0 or 1. When we program later in this assignment, indices will start at zero for consistency with Python indexing.

The "horizontal derivative" is the derivative with respect to $c$. A derivative in this direction is positive when image values increase as we go to greater column indices ("to the right" in the image).

The "vertical derivative" is the derivative with respect to $r$. A derivative in this direction is positive when image values increase as we go to greater row indices ("down" in the image).

When giving a discrete estimate of the gradient of $I$ we give the horizontal derivative $I_c$ first and the vertical derivative $I_r$ second. These are two arrays.

We will use this notation consistently throughout this assignment.

Problem 2.3 (Exam Style)¶

Padding with zeros, as you did in Problem 2.1, amounts to placing a black rim (in images, the pixels that are zero appear as black) on some sides of the input image. This causes artificially high values in the derivatives near image boundaries in most cases.

A better way when differentiating is to pad by replication instead, which amounts to repeating the nearest row or column rather than adding zeros. This typically causes less abrupt derivative values near the boundaries (although this will be hard to see with the random values in the image $I$ for this part).

Example¶

Suppose that we want to pad $I$ with two columns on each side and one row at the top (this is not the case for finite differences, just an example). You can first add the extra row and then add the two sets of extra columns on the result, or vice versa. The result is the same ($I$ is in bold):

Problem 3.1¶

def gaussian(sigma, trunc_multiple=3.5):

and

def d_gaussian(sigma, trunc_multiple=3.5):

that compute a one-dimensional normalized Gaussian kernel and its normalized derivative. Both kernels use the given parameter sigma for the Gaussian and are truncated roughly at $\sigma\tau$ where $\sigma$ is sigma and $\tau$ is trunc_multiple.

More precisely, gaussian computes a numpy vector g of $n$ floating-point samples where $n = 2h + 1$ and $h = \lceil\sigma\tau\rceil$ (the notation $\lceil\cdot\rceil$ denotes the ceil function). The samples are computed and normalized as specified in the class notes. The resulting vector is an averaging (or smoothing) kernel.

The function gaussian returns both g and a numpy vector x equal to np.arange(-h, h+1). This is the vector of coordinates on which the values of the Gaussian are computed.

The function d_gaussian is similar but returns d and x where d is the Gaussian differentiation kernel, computed and normalized as specified in the notes.

def retrieve(file_name, semester='spring24', homework=1):
    if osp.exists(file_name):
        print('Using previously downloaded file {}'.format(file_name))
    else:
        context = ssl._create_unverified_context()
        fmt = 'https://www2.cs.duke.edu/courses/{}/compsci527/homework/{}/{}'
        url = fmt.format(semester, homework, file_name)
        with urllib.request.urlopen(url, context=context) as response:
            with open(file_name, 'wb') as file:
                shutil.copyfileobj(response, file)
        print('Downloaded file {}'.format(file_name))

Problem 4.3¶

Use the interpolation function scipy.interpolate.interpn to write a function with header

def ridges(g, thr):

This function takes a gradient pair g = (gc, gr) for some black-and-white image and a nonnegative threshold thr and returns a binary map of all strong gradient ridge points in the image. These points are defined next.

Let $m$ be the magnitude of the gradient $\mathbf{g}$, and let $t$ be the threshold thr. A pixel at row $r$ and column $c$ is a strong gradient ridge point if it satisfies the following conditions:

  • $m(r, c) \geq t$ and $m(r, c) > 0$.
  • The point $(r, c)$ is at least one pixel away from the nearest image boundary.
  • The gradient magnitude $m(r, c)$ in the direction of the gradient $\mathbf{g}(r, c)$ at $(r, c)$ is a local maximum. This can be determined as follows:
    • Let $\mathbf{p} = (r, c)$.
    • Let $\mathbf{n} = \mathbf{g}(\mathbf{p}) / \|\mathbf{g}(\mathbf{p})\|$ be a unit vector in the direction of the gradient at $\mathbf{p}$. Since $m(\mathbf{p}) > 0$, this vector is well defined.
    • Determine the two points $\mathbf{a} \text{ and } \mathbf{b}$ where $\mathbf{a} = \mathbf{p} + \mathbf{n}$ and $\mathbf{b} = \mathbf{p} - \mathbf{n}$ that are one pixel away from $\mathbf{p}$ in the positive and negative direction of the gradient. These two points are determined by bilinear interpolation.
    • The point $\mathbf{p}$ is a local maximum for $m$ in the direction of the gradient if $m(\mathbf{p}) > m(\mathbf{a})$ and $m(\mathbf{p}) > m(\mathbf{b})$. That is, walking from $\mathbf{a}$ through $\mathbf{p}$ and to $\mathbf{b}$ you first go uphill and then downhill.

Notes¶

  • It may take a bit of patience to parse the definition of strong ridge point above. You may want to make a drawing for yourself to understand what is going on.
  • This problem is more an exercise in numpy than anything else. Use matrix operations without explicit for loops, or your program wil take forever. Here is one possible strategy, but this is not mandatory:
    • Compute the gradient magnitude $m$ as a single array.
    • Find the indices of all the points that are at least one pixel away from the image boundaries and where $m \geq t$. We found the function numpy.flatnonzero useful here.
    • Find two vectors rows and columns with the coordinates of all those points. We found numpy.unravel_index useful here.
    • Flatten the valid parts of $m$, $g_r$, and $g_c$ into vectors using numpy.ravel().
    • Compute the unit vectors $\mathbf{n}$ (all at once, no explicit for loops).
    • Compute the points $\mathbf{a}$ and $\mathbf{b}$ (ditto).
    • Use scipy.interpolate.interpn to find the values of $m$ at those points.
    • The rest is simple.
  • Read the specification of scipy.interpolate.interpn very carefully, paying particular attention to the required shapes of the arguments points and xi. The formats of these two arguments are different, and you may need some combination of numpy functions such as arange, linspace, meshgrid, and/or column_stack to make values to pass to these arguments.
